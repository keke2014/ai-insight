# AI洞察网站内容更新说明文档

## 一、 总述：自动化内容更新管线

AI 洞察网站的内容更新告别了传统的手工编写网页模式，而是采用了一套高度自动化的 **“数据驱动 + 脚本生成 + 一键部署”** 的工程化更新管线。

核心更新流程分为四个阶段：**数据准备 $\rightarrow$ 页面组装 $\rightarrow$ 质量门禁 $\rightarrow$ 自动化部署**。复杂的排版和发布逻辑均由 `scripts/` 目录下的系列脚本接管，内容创作者只需负责提供原始的、结构化的数据文件。

---

## 二、 分述：核心更新环节详解

### 1. 数据准备与调度 (Data Preparation & Orchestration)
网站的每日洞察内容彻底剥离了样式表现，直接以标准化的 JSON 格式进行存储。
- **结构化产出**：大模型（或创作者）每天会生成一份诸如 `data/daily-content-YYYY-MM-DD.json` 的纯数据文件。该文件包含了当天的资讯、摘要与分析，代码行数通常极少，极大地节约了编写的时间。
- **如何生成当天的 JSON (具体操作)**：
  - **默认生成**：如果是全新创作一天，运行 `python scripts/gen_daily_json.py`。会自动帮你在 `data/` 下创建带有今日日期（如 `daily-content-2026-03-22.json`）的标准空白配置。
  - **指定日期生成**：如果您计划为任意未来的一天起草，可带上特定日期以及 `--template`，如：`python scripts/gen_daily_json.py 2026-03-22 --template`
  - **继承复刻**：如果您想要复用昨天的结构（修改局部旧闻从而避免通篇重写），可以使用 `--from` 快捷继承命令，如：`python scripts/gen_daily_json.py 2026-03-22 --from 2026-03-21`。
- **流程编排**：由 `scripts/ai_daily_orchestrator.py` 提供支持日志，它像一个统筹分配系统一样，管理、记录每一次生成任务的上下文与进度状态快照。

### 2. 页面组装与模板渲染 (HTML Generation)
每天的内容准备完毕后，再由专门的脚本为其穿上“网页外衣”：
- **组合生成**：使用 `scripts/gen_daily_html.py <日期>` 命令。该脚本自动读取当天的 JSON，并与 `templates/` 目下的 CSS / JS 设计模板进行组合，最后输出成完整的长篇 HTML 网页并保存到 `01-daily-reports/`。
- **首页自动联动**：除了生成子页面，脚本还会在首页 `index.html` 的最新进展面板和浏览日历上，动态插入新文档的超链接和内容提要。

### 3. 上线前质检门禁 (Quality Gate)
在推送到线上产生实际影响前，系统配置了严格的校验来防御低级失误：
- **安全与合规扫描**：`scripts/daily_quality_gate.py` 会在提交前被触发，强制检查生成的网页是否有失效链接、内部敏感词泄露或者是关键的 JSON 字段缺失（比如“明日关注”栏目是否意外为空）。任何 `Hard Fail` 级别的校验不通过都会被直接拦截。

### 4. 自动化分发与部署 (Automated Deployment & Sync)
一切完备后，系统通过主令脚本进行一站式分发：
- **一键脚本部署**：创作者只需执行主文件：`./scripts/deploy_daily.sh <日期>`。此脚本会依次：
  1. 调用并核实以上的全套生成及门禁流程；
  2. 自动执行 `git add`, `git commit` 以及 `git push` 到在线仓库；
  3. GitHub Pages 即因此触发自动化站点热更新。
- **公私域版本控制**：为了兼顾内外场景需求，脚本还会同时调用 `scripts/sync_to_public.py`，将含私密内容的报告一键脱敏，同步生成到 `public/` 目录下用于外部分享。另外包含 `send_ai_daily.py` 在内的脚本也能一键将资讯图文推送到 KIM 等即时通讯工具的工作群中。

---

## 三、 总结规范：如何执行一次日常更新？

如果您是这套系统的运维者，在通常的发布日里，您无需手动翻找各级标签和代码。只需要：

1. **模板起草**：为当天开辟一份数据输入档（在终端执行：`python scripts/gen_daily_json.py`）
2. **输入阶段**：确保存有当天内容的 JSON 文件位于 `data/` 目录中。
3. **指令下达**：
   ```bash
   ./scripts/deploy_daily.sh 2026-03-22
   ```
4. **完成验收**：剩下的事交给脚本。它会在数十秒内完成从页面生成、漏洞规避、公开版剥离到自动推送到网站的全程运作，您只需最终在线上检阅更新成果。
